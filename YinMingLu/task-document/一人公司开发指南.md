# 《幽冥录》/ Chronicles of Yin - 一人公司开发指南（Godot版）

## 概述

**适用对象**：独立开发者 / 一人公司  
**开发模式**：AI 辅助开发 + 开源资源  
**目标**：6-8 个月完成 MVP（第一章+第二章）  
**预算**：¥30,000 - ¥50,000  
**核心工具**：Godot 4.3 + AI 工具链 + 免费资源

---

## 一、为什么选择 Godot？

### 1.1 与 Unity 对比优势

| 维度 | Godot 4.3 | Unity 2022 |
|------|-----------|------------|
| **授权费用** | 完全免费（MIT） | 个人版免费（有收入限制） |
| **国内合规** | 无风险，正常访问 | 团结引擎分叉，存在不确定性 |
| **引擎体积** | 下载包 ~70MB | 安装包 ~3GB |
| **启动速度** | 秒开 | 较慢 |
| **GDScript** | 类似 Python，易上手 | C# 学习曲线较陡 |
| **内置网络** | 原生多人游戏API | 需第三方库（Mirror/Netcode） |
| **移动端导出** | 一键导出，体积小 | 配置较复杂 |
| **AI辅助支持** | Cursor/Copilot 支持良好 | 支持更好（生态更大） |

### 1.2 本项目选择 Godot 的核心原因

1. **规避授权风险**：Unity 国内政策不确定性，Godot 完全开源安全
2. **局域网联机简单**：内置 Multiplayer API，无需第三方库
3. **小团队友好**：轻量、快速迭代、学习成本低
4. **预算更低**：零授权费，总预算可降低 30-40%
5. **移动端优化**：首包更容易控制在 500MB 以内

## 二、开发理念

### 2.1 核心原则

```
1. 先跑起来，再跑得好
2. 用 AI 做 80%，自己调 20%
3. 免费资源先用，后期再换
4. 每周必须有可玩版本
5. 先做减法，再做加法
```

### 3.2 Godot 关键概念（Unity开发者迁移）

| Unity 概念 | Godot 对应 | 说明 |
|------------|-----------|------|
| GameObject | Node | 基础实体 |
| Component | Node（组合） | Godot 用节点树组合功能 |
| Prefab | PackedScene | 可复用场景 |
| ScriptableObject | Resource (.tres) | 数据资源文件 |
| MonoBehaviour | 继承 Node 的脚本 | 附加到节点上运行 |
| Update() | _process(delta) | 每帧调用 |
| FixedUpdate() | _physics_process(delta) | 物理更新 |
| Start() | _ready() | 初始化 |
| Instantiate() | instantiate() | 实例化场景 |
| Destroy() | queue_free() | 销毁节点 |
| SceneManager | get_tree().change_scene() | 场景切换 |

## 三、AI 工具使用指南

### 3.1 Cursor（代码编写）

**使用技巧**：
```
1. 先描述整体需求，再逐步细化
2. 让 AI 先生成框架，再填充细节
3. 遇到报错直接粘贴给 AI
4. 使用 @ 引用项目中的其他文件
```

**Prompt 模板**：
```
请帮我写一个 [功能名称]，包含：
1. [功能点 1]
2. [功能点 2]
3. [功能点 3]

参考 [游戏/框架] 的实现方式。
使用 [设计模式] 设计。
```

### 3.2 Midjourney（美术概念）

**使用技巧**：
```
1. 先找参考图，描述风格
2. 使用 --seed 保持风格一致
3. 生成多张选最好的
4. 用 Photoshop 简单修改
```

**Prompt 模板**：
```
[主体], [风格], [氛围], [光线], [颜色],
[细节描述], [参考游戏/电影],
--ar [比例] --v 6 --style raw
```

### 3.3 Suno（音乐生成）

**使用技巧**：
```
1. 描述场景氛围
2. 指定乐器（古筝、二胡）
3. 生成多首选最好的
4. 使用 [Extend] 延长音乐
```

**Prompt 模板**：
```
[场景] background music,
[氛围], [乐器],
[情绪], [用途],
loopable, game music
```

### 3.4 Tripo3D（3D 模型）

**使用技巧**：
```
1. 用 Midjourney 生成三视图
2. 上传到 Tripo3D
3. 生成后导入 Blender 修改
4. 减面优化（适合移动端）
```

## 六、常见问题解决

### Q1: AI 生成的代码有 Bug 怎么办？

**解决步骤**：
```
1. 把报错信息粘贴给 Cursor
2. 说 "修复这个报错"
3. 如果还报错，简化需求重新生成
4. 自己学习相关知识点
```

### Q2: 美术资源风格不统一怎么办？

**解决步骤**：
```
1. 确定主色调（暗色调 + 青绿色）
2. 使用 Midjourney 的 --seed 保持一致
3. 后期统一调色（Photoshop 或 Unity Post Processing）
4. 用 Low Poly 风格降低要求
```

### Q3: 游戏性能差怎么办？

**优化步骤**：
```
1. 使用 Unity Profiler 找出瓶颈
2. 减少 Draw Call（静态合批、动态合批）
3. 使用对象池（Object Pooling）
4. 降低贴图分辨率
5. 减少粒子数量
```

**AI Prompt**：
```
请帮我优化 Unity 游戏性能，当前问题：
[描述问题，如：帧率低、卡顿、内存高]

Profiler 显示：
[粘贴 Profiler 数据]

请给出优化建议和代码。
```

### Q4: 联机不同步怎么办？

**解决步骤**：
```
1. 检查 Mirror 的 sync 设置
2. 使用 [Command] 和 [ClientRpc] 正确
3. 减少同步频率（非关键数据 10Hz）
4. 使用插值平滑显示
```
